# Finance Bot -> Google Sheets

Телеграм-бот для учета финансов в свободной форме. Вы пишете сообщения вроде:

- `-3000₽ за amo crm`
- `-100.000₽ себе стоимость`
- `+250000₽ выручка`

Бот:

- распознает сумму, знак, дату и описание;
- понимает естественный язык и относительные даты через OpenAI;
- определяет категорию и PnL-группу на русском языке;
- записывает операцию в Google Sheets;
- использует отдельные листы `Операции`, `ДДС`, `PNL`, `Дашборд`;
- создает базовые графики по денежному потоку и прибыли.

## Что создается в таблице

- `Операции` - все операции по строкам.
- `ДДС` - движение денежных средств по месяцам.
- `PNL` - выручка, себестоимость, операционные расходы и прибыль по месяцам.
- `Дашборд` - ключевые метрики и графики.

## Быстрый запуск

1. Создайте Telegram-бота через `@BotFather` и получите `BOT_TOKEN`.
2. Создайте Google Spreadsheet.
3. Создайте Service Account в Google Cloud и включите Google Sheets API.
4. Поделитесь таблицей с email сервисного аккаунта с правами `Editor`.
5. Скопируйте `.env.example` в `.env` и заполните переменные.
6. Чтобы бот понимал свободные фразы и относительные даты, добавьте `OPENAI_API_KEY`.
7. Установите зависимости:

```bash
npm install
```

8. Инициализируйте структуру таблицы:

```bash
npm run setup:sheets
```

9. Запустите бота:

```bash
npm start
```

## Формат сообщений

Поддерживаются варианты:

- `-3000₽ amo crm`
- `-100 000₽ себестоимость`
- `-15 500 реклама 02.04`
- `+470000 выручка`
- `ровно неделю назад я заплатил менеджеру Игорю 10.000₽`
- `в прошлый понедельник поступило 350000 от клиента`

Если дата не указана, используется текущая. Если задан `OPENAI_API_KEY`, бот сначала пытается разобрать сообщение через OpenAI, а потом при необходимости откатывается к regex-парсеру.

## Логика учета

- положительная сумма -> доход;
- отрицательная сумма -> расход;
- OpenAI понимает свободные фразы, относительные даты и сам выбирает категорию;
- если OpenAI недоступен, ключевые слова вроде `выручка`, `себестоимость`, `amo`, `реклама`, `зарплата`, `налог` помогают определить категорию;
- если категория не распознана, расход идет в `Прочие расходы`, а доход в `Выручка`.

## Важное ограничение

Сейчас это базовая версия MVP:

- один бот;
- одна таблица;
- одна валюта `RUB`;
- fallback-классификация по ключевым словам.

Следующим шагом можно добавить:

- голосовые сообщения;
- отдельные проекты или юрлица;
- подтверждение перед записью;
- команды `/dds`, `/pnl`, `/month`;
- автосоздание новых категорий;
- раздельный учет по сотрудникам, проектам и юрлицам.
